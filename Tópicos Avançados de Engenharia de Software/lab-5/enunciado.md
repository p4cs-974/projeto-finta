# Enunciado

Objetivo: Modelar o processo de negócio TO-BE do sistema utilizando BPMN, com nível de detalhe suficiente para permitir posteriormente a identificação de operações e serviços. Especificar os requisitos do sistema consistentes com o processo de negócio modelado.

## Atividades

- Identificar um processo de negócio do seu sistema. (É importante levar em consideração que o processo tenha:  várias etapas, envolva decisões, manipule dados do sistema, possua regras de negócio e eventualmente envolva integrações externas)
- Modelar o processo TO-BE utilizando BPMN (obrigatório).
- Descrever as atividades do processo: As atividades devem ter nível de detalhe suficiente para representar funcionalidades do sistema.
- Identificar e documentar regras de negócio relacionadas às atividades do processo.
- Elaborar o diagrama de casos de uso (consistente com o processo To Be)
- Especificar 5 requisitos não funcionais consistentes com o comportamento do processo de negócio.

## Entrega 

Elaborar e enviar um documento com as seguintes informações:

- Diagrama BPMN do processo TO-BE com a descrição das tarefas.
- Lista de regras de negócio, no formato: RN01, RN02.
- Diagrama de casos de uso (notação UML)
- Lista de requisitos não funcionais, no formato: RNF01, RNF02.
- Colocar no moodle Link do GIT com as atualizações do projeto
